## Backend Agent

任务：

- [x] 增加 SQLite 初始化和 migration。
- [x] 实现 `source_files` repository。
- [x] 实现 `usage_events` repository。
- [x] 实现 `--ingest` CLI。
- [x] 把 P1 JSONL 解析逻辑复用到 ingestion。
- [x] 实现从 SQLite 聚合 `/api/report`。
- [x] 保持 P1 `/api/report` contract 兼容。

验收：

- [ ] 首次 ingestion 插入 usage events。
- [ ] 二次 ingestion 不重复插入。
- [ ] 新增 JSONL 行后只插入新增 delta。
- [ ] 数据库 schema 与 contract 一致。

## Test Agent

任务：

- [x] 准备脱敏 JSONL fixture。
- [x] 覆盖 cumulative snapshot delta。
- [x] 覆盖重复 ingestion 去重。
- [x] 覆盖 report contract 字段。
- [ ] 覆盖重复 snapshot 跳过。
- [ ] 覆盖文件变小或替换场景。

验收：

- [x] 测试不包含 prompt/response 正文。
- [x] 测试可在无真实 Codex 日志环境运行。

## Frontend Agent

任务：

- [ ] 确认 dashboard 无需字段变更。
- [ ] 如增加 ingestion 状态展示，必须先补 API contract。

验收：

- [ ] 页面仍只消费 `/api/report` contract 字段。
- [ ] 不在前端计算统计口径。

## Review Agent

任务：

- [ ] 检查 schema 是否存敏感内容。
- [ ] 检查 ingestion 是否重复计数。
- [ ] 检查 checkpoint 是否可能跳过未处理行。
- [ ] 检查 `/api/report` 是否兼容 P1。

产出：

- `docs/reviews/P2-2026-04-30-codex-sqlite-review.md`

## Acceptance Agent

任务：

- [ ] 按 P2 acceptance 模板执行。
- [ ] 记录真实终端构建结果。
- [ ] 记录 ingestion 输出。
- [ ] 记录 smoke test 或等价端到端验证结果。

产出：

- `docs/acceptance/P2-2026-04-30-codex-sqlite.md`

## P2 完成定义

P2 完成必须同时满足：

- SQLite schema 初始化成功。
- `--ingest` 可导入真实 Codex usage。
- 重复 ingestion 不重复计数。
- `/api/report` 从 SQLite 返回 P1 contract 结构。
- 构建通过。
- review 通过。
- acceptance 状态为通过或有明确可接受条件。
